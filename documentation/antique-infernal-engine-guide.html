What it does

A field-guide page for any antique

Hand it a photo — a pinwheel calculator, a voltaic pile, a Jacquard punch card — and it produces a single field-guide page that takes the artifact seriously enough to do the arithmetic.

  • Identifies the artifact and its real mechanism.
  • Classifies the path: can it produce power, or can it compute?
  • Calculates — with frozen, deterministic math — the units needed to power one AI hello, plus the equivalent in potatoes and pedaling cyclists.
  • Writes it up in the field-guide voice: absurd but numerate, every joke sitting on a real number.

Every page is reproducible. The prose model never touches arithmetic — it receives frozen math, and only then is it allowed to be funny.

How it works

The adjudication loop

This is not a linear chain. The Reality Badger verifies the math and, on failure, names which earlier agent is at fault. The orchestrator re-dispatches — back to the Artifact Goblin to re-look, or to Sherlock Ohms to re-research — bounded at two loops. That feedback is what makes this a cooperating system instead of a pipeline, and it is fully visible in trace.json.

The demo beat: open trace.json and watch the Badger reject a first pass and send it back to re-classify. The loop count, the route_to, and the reason are all logged. The loop is the evidence.

Quickstart

Runs fully local, with zero keys

Granite runs locally through Ollama. Frontier vision and prose are optional add-ons, not the foundation.

# 1 · clone
$ git clone https://github.com/Angiebio/jun2026-builder-sat
$ cd jun2026-builder-sat

# 2 · install
$ python -m venv .venv && source .venv/bin/activate
$ pip install -r requirements.txt

# 3 · pull the local model
$ ollama pull granite4:micro

# 4 · run on a bundled fixture — no network required
$ python agent.py run --fixture fixtures/pinwheel.jpg

You'll get a field-guide page in runs/<id>/article.md and the full decision trail in runs/<id>/trace.json.

Optional — turn on frontier features

$ cp .env.example .env       # then add your keys
# ANTHROPIC_API_KEY=...        ($50 attendee credits at the event)

Usage

Everyday commands

# run on a bundled fixture (deterministic, fully local)
$ python agent.py run --fixture fixtures/voltaic.jpg

# run on your own antique photo (uses frontier vision if a key is set)
$ python agent.py run --image ~/photos/mystery-gadget.jpg

# serve the single-page web UI at http://localhost:8000
$ python agent.py serve

# run the eval suite, with and without skills
$ python agent.py eval
$ python agent.py eval --no-skill

Every run writes a self-contained folder under runs/<id>/: the input, each agent's JSON, the frozen math, the QC result, the article, and trace.json.

The two skills

One does the math. One holds the voice.

antique-power-math

The deterministic one

The LLM proposes physical assumptions; Python does the arithmetic. Constants are frozen — 30 W per AI hello, 0.0001 W per potato, 100 W per cyclist. Any value at or below zero raises and fails loud: no silent zero, no fabricated unit.

trcl-field-guide-writer

The voice one

Enforces the field-guide schema and tone: absurd but numerate, never hiding uncertainty, one memorable line per page. The writer receives frozen math and never calculates. It can only narrate the answer.

Evals

We measure whether the skill earns its place

Three fixtures — a pinwheel calculator, a voltaic pile, a Jacquard punch card — each run with and without the math skill. Assertions are script-checked, not LLM-judged: the math JSON must exactly match the calculator, QC must pass, and Granite must appear in the trace for both classify and QC.

Without the math skill, the prose model hallucinated units. With it, the calculator matched 3 of 3. A skill that can't beat its --no-skill baseline isn't earning its keep.
