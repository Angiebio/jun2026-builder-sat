What it does

A field-guide page for any antique

Hand it a photo — a pinwheel calculator, a FORTRAN punch-card deck, a TI-82, a Nokia candybar — and it produces a single field-guide page that takes the artifact seriously enough to do the arithmetic.

  • Identifies the artifact and its real mechanism.
  • Classifies the path: can it produce power, or can it compute?
  • Calculates — with frozen, deterministic math — the time or units needed to reach one AI hello, plus the equivalent in potato batteries and pedaling cyclists.
  • Writes it up in the field-guide voice: absurd but numerate, every joke sitting on a real number.

Every page is reproducible. The prose model never touches arithmetic — it receives frozen math, and only then is it allowed to be funny.

The four-specimen arc. Punched cards: never. Pinwheel calculator: 443 years. TI-82: 6.5 hours. Nokia 3590: 10 minutes. The hardware gets younger; the verdict gets faster — same question, same physics, every time.

How it works

The adjudication loop

This is not a linear chain. The Reality Badger verifies the math and, on failure, names which earlier agent is at fault. The orchestrator re-dispatches — back to the Artifact Goblin to re-look, or to Sherlock Ohms to re-research — bounded at two loops. That feedback is what makes this a cooperating system instead of a pipeline, and it is fully visible in trace.json.

The demo beat: open trace.json and read the loop — the Badger's verdict, the route_to, and the loop count are all logged. On the default profile every step honestly records deterministic / fixture: no model in the correctness path. The loop is the evidence.

Quickstart

Runs fully local, with zero keys

The default FLOOR profile is deterministic — fixtures plus Python — so the core demo can never die on venue Wi-Fi or a dead key. Frontier vision and prose are an opt-in add-on, not the foundation.

# 1 · clone
$ git clone https://github.com/Angiebio/jun2026-builder-sat
$ cd jun2026-builder-sat

# 2 · install (uv-managed env — no requirements.txt)
$ uv sync

# 3 · run a bundled fixture — zero network, zero keys
$ uv run python orchestrator.py pinwheel

You'll get a field-guide page in runs/<case>/article.md and the full decision trail in runs/<case>/trace.json.

Optional — turn on --live frontier features

$ cp .env.example .env       # then add your key
# ANTHROPIC_API_KEY=...        ($50 attendee credits at the event)

Usage

Everyday commands

# run any of the four bundled fixtures (deterministic, fully local)
$ uv run python orchestrator.py pinwheel        # mechanical calculator → 443 years
$ uv run python orchestrator.py ti_82           # graphing calculator   → 6.5 hours

# serve the single-page web UI at http://127.0.0.1:8000

# validate both skills against the agentskills.io spec
$ uv run python -m skills_ref.cli validate ./skills/antique-power-math
$ uv run python -m skills_ref.cli validate ./skills/trcl-field-guide-writer

# run the eval suite, with and without the skill
$ uv run python -m evals

Every run writes a self-contained folder under runs/<case>/: each agent's JSON, the frozen math, the QC result, the article, and trace.json.

The two skills

One does the math. One holds the voice.

antique-power-math

The deterministic one

The LLM proposes the path and one assumption; Python does the arithmetic. Constants are frozen — 30 W per AI hello, 0.0001 W per potato, 100 W per cyclist. A negative value fails loud; a zero on a compute path becomes the -1.0 sentinel the writer renders as “Never (∞).” No silent zero, no fabricated unit.

trcl-field-guide-writer

The voice one

Enforces the field-guide schema and tone: absurd but numerate, never hiding uncertainty, one memorable line per page. The writer receives frozen math and never recalculates. It can only narrate the answer.

Both pass agentskills validate against the agentskills.io spec.
